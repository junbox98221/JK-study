# 12. Lanes와 이벤트 우선순위

> 이번 챕터에선 React 18 이후 도입된 Lanes 모델을 통한 이벤트 우선순위 결정 방식을 분석합니다.

React 18 이후부터는 **Lanes** 모델을 사용하여 작업의 우선순위를 관리합니다. Lanes는 비트마스크 기반의 우선순위 시스템으로, 기존의 expirationTime보다 유연하고 효율적입니다.

## 1. Lanes의 개념

Lanes는 32비트 비트마스크로 표현되며, **작은 비트일수록 높은 우선순위**를 가집니다.

```javascript
// react/packages/react-reconciler/src/ReactFiberLane.js

export type Lane = number;
export type Lanes = number;

// 우선순위 예시
const SyncLane = 0b0000000000000000000000000000001;      // 1
const InputContinuousLane = 0b0000000000000000000000000000100; // 4
const DefaultLane = 0b0000000000000000000000000010000;   // 16
const IdleLane = 0b0100000000000000000000000000000;      // 2^30
```

비트마스크를 사용하면 여러 lane을 동시에 표현할 수 있습니다.

```javascript
// 여러 우선순위를 동시에 가질 수 있음
const lanes = SyncLane | DefaultLane; // 0b...10001
```

## 2. Lane 계산 방식

### requestUpdateLane

```javascript
// react/packages/react-reconciler/src/ReactFiberWorkLoop.js

function requestUpdateLane(fiber: Fiber): Lane {
  const mode = fiber.mode;

  if ((mode & ConcurrentMode) === NoMode) {
    return SyncLane; // 동기 모드
  }

  // 현재 실행 중인 transition이 있는지 확인
  const transition = ReactSharedInternals.T;
  if (transition !== null) {
    return requestTransitionLane(transition);
  }

  // 현재 이벤트의 우선순위를 확인
  const updateLane = getCurrentUpdatePriority();
  if (updateLane !== NoLane) {
    return updateLane;
  }

  // 이벤트 우선순위를 기반으로 lane 결정
  return eventPriorityToLane(getCurrentEventPriority());
}
```

Fiber를 위해 lane을 계산하는 함수입니다. 동기 모드인지, transition이 있는지, 현재 이벤트 우선순위가 무엇인지에 따라 적절한 lane을 반환합니다.

### getCurrentEventPriority

```javascript
// react/packages/react-dom-bindings/src/client/ReactDOMHostConfig.js

export function getCurrentEventPriority(): EventPriority {
  const currentEvent = window.event;
  if (currentEvent === undefined) {
    return DefaultEventPriority;
  }

  return getEventPriority(currentEvent.type);
}

function getEventPriority(domEventName: string): EventPriority {
  switch (domEventName) {
    case 'click':
    case 'keydown':
    case 'input':
      return DiscreteEventPriority; // 높은 우선순위

    case 'drag':
    case 'scroll':
      return ContinuousEventPriority; // 중간 우선순위

    default:
      return DefaultEventPriority; // 기본 우선순위
  }
}
```

현재 발생한 이벤트의 타입을 확인하여 우선순위를 결정합니다. 클릭, 키 입력 등은 높은 우선순위를, 스크롤 등은 중간 우선순위를 받습니다.

### eventPriorityToLane

```javascript
// react/packages/react-reconciler/src/ReactEventPriorities.js

export function eventPriorityToLane(eventPriority: EventPriority): Lane {
  switch (eventPriority) {
    case DiscreteEventPriority:
      return SyncLane;

    case ContinuousEventPriority:
      return InputContinuousLane;

    case DefaultEventPriority:
      return DefaultLane;

    case IdleEventPriority:
      return IdleLane;

    default:
      return NoLane;
  }
}
```

이벤트 우선순위를 lane으로 변환합니다.

## 3. Lane 상수값들

```javascript
// react/packages/react-reconciler/src/ReactFiberLane.js

export const NoLanes: Lanes = 0b0000000000000000000000000000000;
export const NoLane: Lane = 0b0000000000000000000000000000000;

export const SyncLane: Lane = 0b0000000000000000000000000000001;

export const InputContinuousHydrationLane: Lane = 0b0000000000000000000000000000010;
export const InputContinuousLane: Lane = 0b0000000000000000000000000000100;

export const DefaultHydrationLane: Lane = 0b0000000000000000000000000001000;
export const DefaultLane: Lane = 0b0000000000000000000000000010000;

export const IdleHydrationLane: Lane = 0b0001000000000000000000000000000;
export const IdleLane: Lane = 0b0010000000000000000000000000000;
```

### 우선순위 정리

| Lane                  | 비트 위치 | 의미                          | 우선순위 |
| --------------------- | --------- | ----------------------------- | -------- |
| **SyncLane**          | 1         | 동기 처리 (최우선)            | 최고     |
| **InputContinuous**   | 4         | 연속 입력 (드래그, 스크롤 등) | 높음     |
| **DefaultLane**       | 16        | 일반 이벤트 (클릭 등)         | 중간     |
| **IdleLane**          | 2^30      | 유휴 상태                     | 낮음     |
| **NoLane**            | 0         | 작업 없음                     | 없음     |

## 4. Lane 비교 및 연산

### 우선순위 비교

```javascript
// react/packages/react-reconciler/src/ReactFiberLane.js

function includesSomeLane(a: Lanes, b: Lanes): boolean {
  return (a & b) !== NoLanes;
}

function isSubsetOfLanes(set: Lanes, subset: Lanes): boolean {
  return (set & subset) === subset;
}

function getHighestPriorityLane(lanes: Lanes): Lane {
  return lanes & -lanes; // 가장 낮은 비트를 추출
}
```

비트 연산을 사용하여 lane을 비교하고 조작합니다. `getHighestPriorityLane`은 가장 낮은 비트(가장 높은 우선순위)를 추출합니다.

### Lane 병합

```javascript
// 여러 lane을 병합
function mergeLanes(a: Lanes, b: Lanes): Lanes {
  return a | b;
}

// 특정 lane 제거
function removeLanes(set: Lanes, subset: Lanes): Lanes {
  return set & ~subset;
}
```

비트 OR 연산으로 lane을 병합하고, 비트 AND NOT 연산으로 특정 lane을 제거합니다.

## 5. expirationTime과의 차이점

| 특징          | expirationTime (구버전)   | Lanes (React 18+)      |
| ------------- | ------------------------- | ---------------------- |
| **표현 방식** | 숫자 (시간 기반)          | 비트마스크             |
| **우선순위**  | 큰 값 = 높은 우선순위     | 작은 비트 = 높은 우선순위 |
| **병합**      | 어려움                    | 비트 OR로 간단         |
| **유연성**    | 단일 우선순위만 표현      | 여러 우선순위 동시 표현 |
| **성능**      | 시간 계산 필요            | 비트 연산으로 빠름     |

```javascript
// expirationTime (구버전)
const priority = 1073741823 - now(); // 계산 필요

// Lanes (현재)
const priority = SyncLane | DefaultLane; // 비트 연산
```

Lanes 모델은 비트마스크를 사용하여 여러 우선순위를 동시에 표현할 수 있고, 비트 연산으로 빠르게 처리할 수 있어 더 효율적입니다.
